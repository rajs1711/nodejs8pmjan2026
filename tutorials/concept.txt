 Q. What is Package.json ?

 package.json is a configuration file used in projects that use Node.js. It contains metadata about the project 
 and manages dependencies, scripts, and settings.
 example:
 {
  "name": "my-app",
  "version": "1.0.0",
  "description": "My first Node.js app",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"No tests yet\""
  },
  "author": "Your Name",
  "license": "ISC",
  "dependencies": {
    "express": "^4.18.2"
  }
}

name → Project name
version → Current version
description → Short info about the project
main → Entry file
scripts → Commands you can run (npm start, etc.)
dependencies → Libraries your project needs
devDependencies → Tools needed only for development

API (Application Programming Interface)
============================================================================
An API (Application Programming Interface) is a set of rules and 
protocols that allows different software applications to 
communicate and share data with one another. 
It acts as a middleman, enabling developers to use 
existing functions or data from other services without 
needing to understand their internal code. 


How an API Works
APIs operate on a request-response cycle: 

Request: A client (like a mobile app) sends a specific request to an endpoint (a specific URL) on a server.
Processing: The API validates the request, checks for security authentication (like API keys or tokens), and forwards it to the server.

Response: The server processes the request and sends the data back through the API to the client, usually in a standardized format like JSON or XML. 

A common analogy is a waiter in a restaurant: You (the client) tell the waiter (the API) what you want from the menu (documentation). The waiter takes your order to the kitchen (the server), which prepares the food and gives it back to the waiter to deliver to your table. 

Common Types of APIs :=>

Public (Open) APIs: Available for any developer to use with few restrictions (e.g., Google Maps Platform).
Private (Internal) APIs: Used exclusively within a company to connect internal systems.


Architecture For API
=====================
REST (Representational State Transfer): The most common style for web APIs. 
It is lightweight, scalable, 
and uses standard HTTP methods like GET, POST, PUT, and DELETE.

REST Key Components
Resources: Key entities exposed by the API, each identified by a unique URI.
HTTP Methods: Used to perform CRUD operations on resources (GET, POST, PUT, DELETE).
Representations: Data format representing resource state (e.g., JSON, XML).
Status Codes: Indicate the status of a request (e.g., 200 OK, 404 Not Found).
